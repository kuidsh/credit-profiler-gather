Perfilador Express · Guía de Base de Datos

Cómo consultar, revisar y depurar los datos del wizard en desarrollo local

Herramientas disponibles
Recomendada

Prisma Studio

Interfaz visual en el navegador para ver, filtrar, editar y eliminar registros de todas las tablas. La opción más cómoda para desarrollo.

npm run db:studio

Abre en localhost:5555

SQL directo

SQLite CLI

Consultas SQL directas sobre el archivo prisma/dev.db. En Windows también se puede usar DB Browser for SQLite (GUI gratuita, sin instalación de consola).

sqlite3 prisma/dev.db
HTTP

API REST local

Endpoints del servidor Express para consultar clientes directamente desde el navegador o Postman. El servidor debe estar corriendo.

npm run server

Puerto localhost:3001

Las 4 tablas
sesiones_wizard

Cada ejecución completa del Paso 4
Campos principales
  • id — UUID, clave primaria
  • sesionToken — token único de sesión (64 chars)
  • guardarCompleto — true si Banco | Financiera
  • clasificacionRecomendada — Banco | Financiera | Subprime
  • viabilidadInicial — Alta | Media | Baja
  • cargaFinanciera — Cómoda | Justa | Apretada
  • paso5ContactoGuardado — true al guardar sub-paso 1
  • paso5Completado — true al terminar los 3 sub-pasos
  • fuentePersistencia — "completa" | "anonima"
  • creadoEn / actualizadoEn
Relaciones
  • 1:1 → perfiles_completos (opcional)
  • 1:1 → interacciones_anonimas (opcional)
perfiles_completos

Solo si clasificación = Banco o Financiera
Campos principales
  • sesionId — FK a sesiones_wizard
  • 16 variables del wizard (pasos 1–3) en campos individuales
  • ocupacion, ingresoMensual, deudasMensuales, enganche…
  • viabilidadInicial, clasificacionRecomendada
  • capacidadPago, cargaFinanciera
  • porQue, fraseVendedor — texto libre del LLM
  • porcentajeCargaCalculado — decimal calculado
Relaciones
  • Pertenece a → sesiones_wizard
  • 1:1 → datos_personales_paso5 (opcional)
datos_personales_paso5

Cuando usuario completa sub-paso 1 (Contacto)
Campos por sub-paso
  • Sub-paso 1 (siempre): nombres, apellidoPaterno, apellidoMaterno?, telefonoCelular
  • Sub-paso 2 (opcional): fechaNacimiento, rfc?, curp?, genero, estadoCivil, correoElectronico
  • Sub-paso 3 (opcional): calle, colonia, municipioDelegacion, estadoRepublica, codigoPostal
  • subPasoCompletado — 1 | 2 | 3
  • completado — true solo cuando los 3 sub-pasos son completos
Nota
  • Contiene PII — sujeto a LFPDPPP. Nunca exponer en logs.
interacciones_anonimas

Solo si clasificación = Subprime
Campos principales
  • sesionId — FK a sesiones_wizard
  • Solo campos categóricos: ocupacion, compruebaIngresos, historialCrediticio, tipoUnidad…
  • rangosAntiguedad — rango ("1-3 años"), nunca valor exacto
  • rangoIngresos — rango ("15k-25k"), nunca valor exacto
  • rangoEnganche — rango ("10-20%"), nunca valor exacto
  • viabilidadInicial, capacidadPago, cargaFinanciera
Nota
  • Sin PII, sin datos financieros exactos. Solo información categórica.
Diagrama de relaciones
sesiones_wizard
├──
perfiles_completos
0..1 — solo si guardarCompleto = true (Banco | Financiera)
└──
datos_personales_paso5
0..1 — solo si usuario completó sub-paso 1
└──
interacciones_anonimas
0..1 — solo si guardarCompleto = false (Subprime)

Cada SesionWizard tiene exactamente una de las dos ramas activas: o bien perfiles_completos (con posible datos_personales_paso5), o bien interacciones_anonimas — nunca ambas.

Comandos útiles
Comando Para qué sirve
npm run db:studio Abre Prisma Studio en localhost:5555 (GUI visual recomendada)
npm run db:push Aplica cambios del schema al archivo dev.db sin generar migración
npm run db:generate Regenera el cliente Prisma en node_modules (correr tras cambiar schema)
sqlite3 prisma/dev.db Abre la consola SQLite interactiva sobre la base de datos local
.tables Lista todas las tablas (dentro del CLI de SQLite)
.schema sesiones_wizard Muestra el DDL completo de la tabla (dentro del CLI de SQLite)
SELECT * FROM sesiones_wizard ORDER BY "creadoEn" DESC LIMIT 10; Las 10 sesiones más recientes
SELECT * FROM perfiles_completos ORDER BY "creadoEn" DESC LIMIT 5; Los 5 perfiles completos más recientes (Banco / Financiera)
SELECT * FROM datos_personales_paso5 ORDER BY "creadoEn" DESC LIMIT 5; Los 5 registros de datos personales más recientes
SELECT * FROM interacciones_anonimas ORDER BY "creadoEn" DESC LIMIT 5; Las 5 interacciones anónimas más recientes (Subprime)
DELETE FROM datos_personales_paso5 WHERE id='...'; Eliminar un registro de datos personales por ID (útil en pruebas)
Endpoints API para revisar datos
Método URL Descripción
GET http://localhost:3001/api/clientes/recientes Retorna los 10 clientes más recientes de datos_personales_paso5
GET http://localhost:3001/api/clientes?telefono=XXXXXXXXXX Busca un cliente por número de teléfono celular exacto
POST http://localhost:3001/api/guardar-contacto Crea o actualiza (UPSERT) el contacto del sub-paso 1 del Paso 5
PUT http://localhost:3001/api/guardar-perfil/:sesionId Actualiza el perfil con los datos de los sub-pasos 2 y 3 del Paso 5
Estrategia de persistencia
Clasificación
Banco o Financiera
  • guardarCompleto = true
  • sesiones_wizard (siempre)
  • perfiles_completos (con las 16 variables + resultado LLM)
  • datos_personales_paso5 (si el usuario avanza al Paso 5)
Guardado progresivo: sub-paso 1 se guarda inmediatamente via POST /api/guardar-contacto para no perder el contacto si el usuario abandona.
Clasificación
Subprime
  • guardarCompleto = false
  • sesiones_wizard (siempre)
  • interacciones_anonimas (solo rangos, sin PII, sin valores exactos)
NO se crea perfil_completo ni se solicitan datos personales. El Paso 5 no se muestra al vendedor cuando la clasificación es Subprime.
Nota sobre entornos
Dev local

SQLite

  • Archivo:prisma/dev.db
  • Schema:prisma/schema.dev.prisma
  • Proveedor:sqlite
  • Migrations:db push (sin archivos de migración)
  • Requisitos:Solo Node.js — sin Docker ni servidor externo
Producción (AWS)

PostgreSQL Aurora Serverless v2

  • Schema:prisma/schema.prisma
  • Proveedor:postgresql
  • Conexión:DATABASE_URL en Amplify env vars
  • Migrations:prisma migrate deploy (en CI/CD)
  • Desplegado via:AWS Amplify + serverless-http (Lambda)
